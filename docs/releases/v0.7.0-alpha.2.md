# v0.7.0-alpha.2 Release Notes

`v0.7.0-alpha.2` focuses on CI stability and release-integrity hardening.
It builds on the `v0.7.0-alpha.1` runtime-controls foundation (budget/tracing/webhook durability) and SharePoint connector path without introducing new HTTP contract changes.

## Highlights

- Stabilized kind setup in `deploy-smoke` by replacing action-based install with checksum-verified script install + retry.
- Extended release verification from "latest release only" to strict historical sweep mode (`latest-count=10`).
- Added release metadata drift checks (semver prerelease suffix vs GitHub `prerelease` flag parity).
- Added GA promotion gate that requires a successful `release-verify` run on the same commit for GA tags.
- Published offline operator runbook for detached signature and SHA verification.

## Included Components

- CI stability:
  - `.github/workflows/deploy-smoke.yml`
  - `deploy/kind/install-kind.sh`
- Release integrity:
  - `scripts/check_release_assets.py`
  - `.github/workflows/release-verify.yml`
  - `scripts/check_ga_release_gate.py`
  - `.github/workflows/release.yml`
- Operator guidance:
  - `docs/operations/offline-evidence-signature-verification.md`

## Migration Notes from v0.7.0-alpha.1

### 1) release-verify now includes historical strict sweep

`release-verify` validates:
- latest release strict integrity/signature checks
- latest 10 releases strict sweep + prerelease parity checks

### 2) GA tags now enforce same-commit release-verify success

For GA tags (`vX.Y.Z`), release workflow fails unless `release-verify` has a successful run on the exact tag commit SHA.

Prerelease tags (`vX.Y.Z-*`) bypass this gate by design.

### 3) deploy-smoke kind bootstrap path changed

The workflow no longer uses `helm/kind-action@v1`; it now uses repository-managed installer logic with checksum verification retries.

## Compatibility

- No breaking HTTP API changes.
- Runtime-controls stack from `v0.7.0-alpha.1` remains the baseline (response redaction, budget enforcement, tracing, webhook durability).
- SharePoint retrieval path remains supported as shipped in `v0.5.0-alpha.1` + managed identity expansion in `v0.6.0-alpha.1`.
- Existing contracts (`policy`, `audit`, `citations`, `evidence bundle`) remain backward-compatible.

## Known Limitations

- Historical sweep validates the latest 10 releases by default; older releases outside this window require explicit tag checks.
- GA gate depends on GitHub Actions run visibility for commit SHA and workflow naming consistency.
