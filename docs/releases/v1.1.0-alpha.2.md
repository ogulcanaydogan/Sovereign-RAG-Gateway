# v1.1.0-alpha.2 Release Notes

Release date: 2026-03-08

## Summary

`v1.1.0-alpha.2` is a hardening-only prerelease on the path to `v1.1.0` GA.

This cut keeps runtime behavior stable and focuses on release-evidence discipline.

## Highlights

1. Release-track continuity:
- Continued the `v1.1.0` reliability/SLO line with no HTTP API expansion.
- Kept deterministic overload shed behavior and reliability thresholds unchanged.

2. Evidence-first GA preparation:
- Added a single-source `Next (v1.1.0)` roadmap section with measurable closeout criteria.
- Kept strict release-integrity checks (latest + latest10 + evidence contract) as required controls.

3. Version alignment:
- Updated project/app/chart/Terraform defaults to `1.1.0-alpha.2` / `v1.1.0-alpha.2`.

## Compatibility

- HTTP endpoints remain unchanged:
  - `POST /v1/chat/completions`
  - `POST /v1/embeddings`
  - `GET /v1/models`
- Existing error envelope and contract set remain unchanged.

## Migration Notes from v1.1.0-alpha.1

- No configuration migration is required.
- Continue enforcing prerelease minimum gates before tagging:
  - `release-verify` green
  - `slo-reliability` green

## Known Limitations

- Inflight shedding state remains in-process (no distributed coordination backend).
- Reliability suites are deterministic synthetic profiles; complement with environment-specific load tests.
- Weekly evidence automation may require manual PR creation when repository action permissions block bot PR creation.
