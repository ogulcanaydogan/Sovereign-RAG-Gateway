# v1.1.0-alpha.1 Release Notes

Release date: 2026-03-07

## Summary

`v1.1.0-alpha.1` starts the reliability/SLO hardening track after `v1.0.0` GA.

This cut is hardening-only. HTTP API surface remains unchanged.

## Highlights

1. Reliability/SLO gate foundation:
- Added deterministic SLO gate script (`scripts/check_slo_reliability.py`) with explicit thresholds:
  - `error_rate <= 1.0%`
  - `p95_regression_vs_baseline <= 10%`
  - `nominal_shed_rate <= 2.0%`
- Added dedicated `slo-reliability` workflow for PR/manual/scheduled runs.
- Wired reliability gate execution into CI (`.github/workflows/ci.yml`).

2. Deterministic reliability artifacts:
- Added fault-injection suite runner (`scripts/run_fault_injection_suite.py`) for:
  - provider throttling/error storm
  - OPA outage/timeout path
  - budget backend transient failure path
- Added soak profile scenario (`soak_mixed_tenant_30m`) in benchmark runner and standardized soak summary output.

3. Runtime backpressure and overload signaling:
- Added in-process global + tenant in-flight guard (`app/services/inflight_guard.py`).
- Added deterministic overload shed behavior:
  - HTTP `503`
  - `error.code = "overload_shed"`
- Added overload audit field (`overload_shed_reason`) and overload shed metric (`srg_overload_shed_total`).

4. Weekly evidence integration:
- Weekly evidence workflow now generates and consumes:
  - governance benchmark summary
  - fault summary
  - soak summary
  - SLO reliability summary
- Weekly markdown report includes `Reliability/SLO Summary` section with threshold table.

## Compatibility

- HTTP endpoints are unchanged:
  - `POST /v1/chat/completions`
  - `POST /v1/embeddings`
  - `GET /v1/models`
- Existing error envelope structure is unchanged.
- Audit contract is backward-compatible; `overload_shed_reason` is optional.

## Migration Notes from v1.0.0

- For load shedding controls, optionally configure:
  - `SRG_INFLIGHT_GLOBAL_LIMIT`
  - `SRG_INFLIGHT_TENANT_DEFAULT_LIMIT`
  - `SRG_INFLIGHT_TENANT_LIMITS`
- Keep release-integrity guardrails active before tagging prereleases:
  - `release-verify` green
  - `slo-reliability` green

## Known Limitations

- Load shedding is currently in-process only (no distributed inflight state backend).
- Soak/fault suites are deterministic synthetic profiles and should be complemented with environment-specific load tests.
- SLO gate currently evaluates artifact outputs, not live production telemetry.
