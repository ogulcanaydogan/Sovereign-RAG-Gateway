# v0.7.0 Release Notes

`v0.7.0` is the GA release of the v0.7 hardening track.

## Highlights

- Graduated release-integrity controls from prerelease to GA:
  - strict asset integrity verification
  - detached signature verification with published release public key
  - prerelease metadata parity checks
  - historical sweep across latest 10 releases
- Enforced GA publish gate:
  - GA tags (`vX.Y.Z`) require same-commit `release-verify` success
- Added explicit GA readiness workflow:
  - required workflow inventory validation
  - release-sweep artifact output for operator auditability

## Included Components

- Workflows:
  - `.github/workflows/release.yml`
  - `.github/workflows/release-verify.yml`
  - `.github/workflows/ga-readiness.yml`
- Scripts:
  - `scripts/check_release_assets.py`
  - `scripts/check_ga_release_gate.py`
  - `scripts/check_required_workflows.py`
- Ops docs:
  - `docs/operations/offline-evidence-signature-verification.md`

## Compatibility

- No breaking HTTP API changes.
- Policy/audit/citation/evidence contracts remain backward-compatible.

## Migration Notes from v0.7.0-rc1

- Update deployment defaults to `0.7.0` / `v0.7.0`.
- GA release process now expects same-commit `release-verify` proof before `vX.Y.Z` tagging.
