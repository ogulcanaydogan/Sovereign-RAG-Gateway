# v1.1.0 Release Notes

Release date: 2026-03-08

## Summary

`v1.1.0` promotes the reliability/SLO hardening track to GA.

This GA cut is hardening-only and keeps HTTP APIs and contracts backward-compatible.

## Highlights

1. Reliability/SLO baseline is GA:
- Deterministic SLO gate and fault/soak evidence are part of standard operations.
- Deterministic overload shedding behavior remains enforced (`503`, `error.code=overload_shed`).

2. GA promotion safety:
- Same-commit `release-verify` success is required for GA tag publication.
- `check_ga_release_gate.py` enforces this in release workflow.

3. Release integrity continuity:
- Strict latest release verification (bundle integrity + signature + public key).
- Latest-10 historical release integrity sweep.
- Release evidence contract drift checks remain active.

## Compatibility

- HTTP endpoints unchanged:
  - `POST /v1/chat/completions`
  - `POST /v1/embeddings`
  - `GET /v1/models`
- Existing error envelope unchanged.
- `docs/contracts/v1/*` remains backward-compatible.

## Migration Notes from v1.1.0-rc1

- No migration required.
- Existing reliability and release-integrity workflows continue unchanged.

## Known Limitations

- Inflight shedding state remains in-process.
- Reliability suites are synthetic; production environments should add environment-specific live validation.
