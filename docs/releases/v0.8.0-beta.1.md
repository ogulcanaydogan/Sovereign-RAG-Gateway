# v0.8.0-beta.1 Release Dossier

Release date: 2026-03-07

## Summary

`v0.8.0-beta.1` is the beta cut for the hardening-only path to `v1.0.0`.

This release does not introduce new HTTP API features. It consolidates and stabilizes runtime governance, release integrity, and operational evidence workflows that were implemented through the v0.8 alpha line.

## Included in Beta Baseline

1. Runtime governance controls remain active and test-covered:
- fail-closed policy behavior
- tenant budget enforcement (including streaming guardrails)
- input/output redaction accounting
- webhook best-effort delivery with durability path
- trace capture and diagnostics chain

2. Release integrity gates remain strict:
- latest release evidence verification
- historical latest-10 integrity sweep
- detached signature + public key validation
- prerelease flag parity enforcement

3. Operator evidence automation remains enabled:
- weekly evidence report generation
- release verification snapshot assets (`PNG + JSON`)
- rollback drill workflow with report artifacts

## Compatibility

- HTTP API: unchanged.
- Contracts under `docs/contracts/v1/`: backward-compatible.
- Deployment defaults updated to beta version only.

## Known Limitations

- Rollback drill still uses fixed reference tags for baseline drill (`v0.7.0 -> v0.6.0`).
- Stabilization-window checker operates on GitHub Actions history window and depends on workflow run availability.
- Evidence snapshot output is summary-oriented; full dashboard rendering remains external.
