# v1.0.0 Release Notes

Release date: 2026-03-07

## Summary

`v1.0.0` is the GA release of Sovereign RAG Gateway's hardening-only track.

This release focuses on runtime governance reliability and release evidence integrity. It does not introduce breaking HTTP API changes.

## GA Highlights

1. Runtime governance baseline is stabilized:
- policy-first fail-closed request path
- tenant-scoped budget enforcement (including streaming controls)
- input/output redaction accounting in audit records
- best-effort webhook delivery with dead-letter and replay support
- traceability chain across request lifecycle

2. Release integrity controls are enforced at GA level:
- strict latest release verification
- strict historical latest-10 integrity sweep
- detached signature/public key verification
- release evidence contract drift checks
- GA tag gate requiring same-commit `release-verify` success

3. Operational evidence baseline remains active:
- scheduled weekly evidence report generation
- release verification snapshots (`JSON + PNG`)
- rollback drill workflow and operator runbook

## Compatibility

- HTTP endpoints remain unchanged:
  - `POST /v1/chat/completions`
  - `POST /v1/embeddings`
  - `GET /v1/models`
- Contracts in `docs/contracts/v1/` remain backward-compatible.

## Migration Notes from v0.9.0-rc1

- Update deployment defaults to `1.0.0` / `v1.0.0`.
- Keep `release-verify` workflow healthy before any future GA tag pushes.
- Continue using weekly evidence reports as operational audit evidence.

## Known Limitations

- Redaction remains regex/pattern-first and is not a perfect PHI detector.
- Weekly snapshot artifacts are summary outputs, not full dashboard exports.
- Rollback drill currently validates a controlled path and is not fully parameterized for arbitrary version pairs.
