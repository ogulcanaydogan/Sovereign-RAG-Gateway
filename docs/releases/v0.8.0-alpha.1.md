# v0.8.0-alpha.1 Release Dossier

Release date: 2026-03-06

## Summary

`v0.8.0-alpha.1` closes the five-item operational backlog opened under `Next (v0.8.0-alpha.1)` with no HTTP API contract changes.

Focus of this release:

- release-integrity hardening automation,
- operator-visible evidence artifacts,
- rollback readiness validation in kind.

## Delivered (5/5)

1. Stabilization-window evidence automation
- Script: `scripts/check_stabilization_window.py`
- CI wiring: `.github/workflows/ga-readiness.yml`
- Output: JSON pass/fail summary for `deploy-smoke`, `release-verify`, `ci`, `terraform-validate`

2. GA gate integration-like validation
- Extended tests: `tests/unit/test_check_ga_release_gate.py`
- Covers:
  - prerelease bypass for `vX.Y.Z-*`
  - GA failure when same-commit `release-verify` is absent
  - GA success when same-commit `release-verify` exists

3. Release-evidence drift contract checker
- Script: `scripts/check_release_evidence_contract.py`
- CI wiring: `.github/workflows/release-verify.yml`
- Checks:
  - required evidence assets present
  - bundle SHA-256 parity
  - detached signature validity
  - metadata-to-asset naming consistency

4. Weekly operator-facing release verification snapshots
- Script: `scripts/generate_release_verification_snapshot.py`
- Weekly workflow integration: `.github/workflows/weekly-evidence-report.yml`
- Outputs:
  - `docs/benchmarks/reports/assets/release-verification/weekly-YYYY-MM-DD.json`
  - `docs/benchmarks/reports/assets/release-verification/weekly-YYYY-MM-DD.png`

5. One-command rollback drill
- Script: `deploy/kind/rollback-drill.sh`
- Workflow: `.github/workflows/rollback-drill.yml`
- Runbook: `docs/operations/rollback-drill.md`
- Validates: `v0.7.0` deployment -> rollback to `v0.6.0` with endpoint smoke and report generation.

## Compatibility

- HTTP API: unchanged.
- Existing gateway behavior: unchanged for request/response contracts.
- New capabilities are additive and operational.

## Known Limitations

- Rollback drill currently targets fixed tags (`v0.7.0 -> v0.6.0`); generalized semver rollback targeting is not yet implemented.
- Weekly snapshot PNG is intentionally lightweight and summary-oriented (not a full dashboard renderer).
- Stabilization-window checker currently reads up to the latest 100 completed runs per workflow.
