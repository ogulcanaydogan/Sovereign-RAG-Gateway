# v0.7.0-rc1 Release Notes

`v0.7.0-rc1` is the release-candidate checkpoint for GA hardening.

## Highlights

- Added GA readiness workflow to continuously verify release integrity posture:
  - required workflow presence
  - historical release sweep (`latest-count=10`)
  - parity check (`tag prerelease suffix` vs GitHub `prerelease` flag)
- Added required-workflow validation script:
  - `scripts/check_required_workflows.py`
- Kept GA publish guard in release pipeline:
  - GA tags (`vX.Y.Z`) require same-commit `release-verify` success via `scripts/check_ga_release_gate.py`

## Included Components

- Workflow:
  - `.github/workflows/ga-readiness.yml`
- Scripts:
  - `scripts/check_required_workflows.py`
  - `scripts/check_release_assets.py` (historical sweep + parity checks retained)
  - `scripts/check_ga_release_gate.py`
- Tests:
  - `tests/unit/test_check_required_workflows.py`

## Compatibility

- No HTTP API changes.
- No contract-breaking schema changes.
- Existing release evidence assets and verification flow remain intact.

## Known Limitations

- Stabilization evidence still depends on repeated successful CI runs over the target window.
- Legacy compatibility thresholds remain enabled for older tags:
  - evidence gap: `< v0.3.0`
  - missing public key: `< v0.7.0-alpha.1`
